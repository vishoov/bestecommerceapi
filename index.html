Architecture Overview

The platform is built using a microservices architecture, where each core feature is encapsulated in its own service. This ensures scalability, maintainability, and independent deployment.

  • API Gateway: Central entry point for all client requests.
  • Authentication Service: Handles user registration, login, JWT-based authentication, and password management.
  • Product Service: Manages product CRUD, search, and inventory.
  • Cart Service: Manages user carts and cart operations.
  • Order Service: Handles order placement, tracking, and cancellation.
  • Review Service: Manages product reviews and ratings.

API Endpoints

  • Products: POST /createProduct, GET /product, PUT /updateProduct, DELETE /deleteProduct, GET /searchProduct
  • Auth: POST /signup, POST /login, POST /logout, PUT /updatePassword, GET /profile
  • Cart: POST /addToCart, DELETE /deleteFromCart, GET /cart
  • Orders: POST /placeOrder, DELETE /cancelOrder, GET /trackOrder
  • Reviews: POST /createReview, GET /reviews/:productId, DELETE /deleteReview

Security & Best Practices

  • JWT for stateless authentication
  • Password hashing with bCrypt
  • Role-based access control
  • Environment variables for sensitive data (dotenv)
  • Input validation with Mongoose schemas
  • Email notifications for order confirmations and tracking
